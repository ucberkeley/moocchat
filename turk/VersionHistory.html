Version Information

This distributable contains the Amazon Mechanical Turk Command Line Tools version 1.3.1.

This version includes the Amazon Mechanical Turk SDK for Java version 1.2.2.

This version is compatible with Amazon Mechanical Turk web service version 2007-06-21.

The Amazon Mechanical Turk team would appreciate any comments and feedback on the Command Line Tools. Please submit issues via the Amazon Mechanical Turk discussion forums.

    Version 1.1.0 - September 13th, 2007

    • New Feature: Added Qualification Management commands allowing you to create and manage qualifications for your HITs.
    • New Feature: Added an Update HITs command allowing you to update properties of HITs that are in production.
    • New Feature: Added Block Worker and Unblock Worker commands to prevent workers from working on your HITs.
    • New Feature: Support for Developer Sandbox throttling. The tools will auto retry requests if the Developer Sandbox throttles your requests.
    • New Feature: Updated to use the Java SDK 1.1.1 and Requester WSDL version 2007-06-21.

    Version 1.0.1 - June 15th, 2007

    • New Feature: When retrieving results, the data is streamed directly to the Results file instead of being cached in memory and written at once. Avoids out of memory errors when retrieving large data sets.
    • New Feature: The generated Results file now orders the question/answer pairs based on the order they appear in the QuestionForm instead of being ordered alphabetically.
    • New Feature: Progress indicator when results are being retrieved from the system.
    • Fixed: Batch operations halt when the first error is encountered instead of skipping and continuing.
    • Fixed: Removed some extraneous files from the distribution.

    • New Feature: Added several convenience scripts in sample applications to complete HIT workflow (Create HIT->Get Results->Approve And Delete Assignments).
